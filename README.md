# CSES Problem Set

I started solving CSES problems to improve my problem-solving and get better at competitive programming. While solving these problems, I tried to understand the logic behind each solution instead of just memorizing approaches.

## Week 1

This week I solved introductory and recursion-based problems. Most of my time went into understanding how recursion works because I wasn't very confident with it before. I also revised some basic C++ concepts while solving these questions.

## Week 2

I moved on to sorting, searching and greedy problems. I used `vector`, `set`, `multiset` and `map` in many questions and learned when each one is useful. I also got a few TLEs, so I had to rethink my approach and optimize my solutions.

## Week 3

This week I started Dynamic Programming. It was the hardest topic for me. Initially, I found it difficult to identify DP states, but after solving more problems like Dice Combinations, Minimizing Coins and Book Shop, I became more comfortable with writing DP solutions.

## What I learned

* Improved my understanding of recursion and backtracking.
* Learned greedy techniques used in competitive programming.
* Became more comfortable using STL containers.
* Started understanding Dynamic Programming and how to build DP states.
* Learned to pay more attention to constraints before deciding on an approach.

Overall, solving these problems helped me improve both my coding speed and the way I think before writing code.
